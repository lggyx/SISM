# 二、系统设计

## 2.1 系统概述

学生信息管理系统（SISM）采用经典的 MVC（Model-View-Controller）分层架构，基于 Java Servlet 和 JSP 技术实现。系统不依赖数据库，使用内存中的 ArrayList 进行数据存储，降低了部署复杂度，便于学习和演示 Web 核心逻辑。

### 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | HTML + CSS + JSP | JSP 负责页面渲染，CSS 负责样式美化 |
| 控制层 | Jakarta Servlet | 处理 HTTP 请求，进行业务逻辑调用 |
| 模型层 | Java POJO + ArrayList | Student 实体类，StudentDataStore 单例存储 |
| 构建工具 | Maven | 项目依赖管理和打包 |
| 服务器 | Tomcat 9.0+ | Jakarta EE 10 兼容的 Servlet 容器 |

## 2.4 数据库设计

### 2.4.1 E-R 图设计

由于本项目采用内存存储（ArrayList），未使用关系型数据库。以下为模拟的 E-R 设计，若后续迁移至数据库可直接对应：

```
+------------------+         +------------------+
|     管理员        |         |      学生         |
+------------------+         +------------------+
| PK 用户名         |         | PK 学号           |
|    密码           |         |    姓名           |
+------------------+         |    性别           |
                             |    年龄           |
                             |    班级           |
                             +------------------+
```

**说明**：
- 管理员实体：仅预设一个管理员账号，用于系统登录验证。
- 学生实体：包含学号（主键）、姓名、性别、年龄、班级五个属性。

### 2.4.2 数据表设计（模拟）

#### 表 1：admin（管理员表）

| 字段名 | 数据类型 | 长度 | 是否为空 | 说明 |
|--------|----------|------|----------|------|
| username | VARCHAR | 50 | 否 | 主键，用户名 |
| password | VARCHAR | 50 | 否 | 登录密码 |

#### 表 2：student（学生表）

| 字段名 | 数据类型 | 长度 | 是否为空 | 说明 |
|--------|----------|------|----------|------|
| student_id | VARCHAR | 6 | 否 | 主键，学号（6位数字） |
| name | VARCHAR | 50 | 否 | 学生姓名 |
| gender | VARCHAR | 4 | 否 | 性别（男/女） |
| age | INT | - | 否 | 年龄（正整数） |
| clazz | VARCHAR | 50 | 否 | 班级名称 |

## 2.5 时序图设计

### 2.5.1 登录验证时序图

     |                  |-------------------forward------------>|
     |<-----------------响应登录页面----------------------------|
     |                  |                    |                  |

     |                  |                    |                  |

     |----GET /index---->|                   |                  |
