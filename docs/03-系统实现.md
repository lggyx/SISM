## 3.2 主要代码说明

### 3.2.1 实体类 Student.java

该类定义了学生的五个属性：学号、姓名、性别、年龄、班级。实现 `Serializable` 接口以支持 Session 存储。包含全参构造方法、无参构造方法、getter/setter 方法以及 `toString()` 方法。

```java
public class Student implements Serializable {
    private String studentId;   // 学号（6位数字）
    private String name;        // 姓名
    private String gender;      // 性别（男/女）
    private int age;            // 年龄
    private String clazz;       // 班级
    // 构造方法、getter/setter、toString...
}
```

### 3.2.2 数据存储类 StudentDataStore.java

采用**单例模式**（饿汉式延迟加载 + 同步锁），确保整个应用中只有一个数据存储实例。内部使用 `ArrayList<Student>` 存储数据，提供增删改查方法。构造方法中预置了 3 条示例数据，便于直接测试。

```java
public class StudentDataStore {
    private static StudentDataStore instance;
    private final List<Student> students;
    
    public static synchronized StudentDataStore getInstance() {
        if (instance == null) instance = new StudentDataStore();
        return instance;
    }
    // getAllStudents / getStudentById / addStudent / updateStudent / deleteStudent
}
```

### 3.2.3 登录验证 LoginServlet.java

- **doGet**：转发到 `login.jsp` 登录页面。
- **doPost**：接收用户名和密码，与预设值（admin / 123456）比对。验证成功则在 Session 中存入 `user` 属性并重定向到首页；验证失败则将错误信息存入 request 域，转发回登录页显示。

```java
if (ADMIN_USERNAME.equals(username) && ADMIN_PASSWORD.equals(password)) {
    session.setAttribute("user", username);
    response.sendRedirect(request.getContextPath() + "/index");
} else {
    request.setAttribute("errorMsg", "账号或密码错误");
    request.getRequestDispatcher("/login.jsp").forward(request, response);
}
```

### 3.2.4 添加学生 AddStudentServlet.java

- **表单验证**：通过 `validateStudent()` 静态方法统一校验学号格式（6位数字）、必填项非空、年龄为正整数。
- **学号唯一性**：调用 `StudentDataStore.isStudentIdExists()` 检查学号是否重复。
- **错误回显**：验证失败时，将用户已填写的数据回传到页面，避免重复输入。
- **成功跳转**：添加成功后使用 `sendRedirect` 重定向到首页，防止表单重复提交。

### 3.2.5 修改学生 EditStudentServlet.java & UpdateStudentServlet.java

- **EditStudentServlet**：接收 `id` 参数，查询对应学生信息存入 request 域，转发到 `editStudent.jsp` 显示原有数据。
- **UpdateStudentServlet**：接收修改后的表单数据，进行表单验证，调用 `StudentDataStore.updateStudent()` 更新数据，成功后重定向到首页。

### 3.2.6 删除学生 DeleteStudentServlet.java

接收 `id` 参数，调用 `deleteStudent()` 从 ArrayList 中移除对应记录，删除成功后重定向到首页刷新列表。前端通过 JavaScript `confirm()` 函数在点击删除按钮时弹出确认对话框。

### 3.2.7 前端页面关键逻辑

**index.jsp**：
- 使用 JSP 脚本元素 `<% %>` 和 `<%= %>` 遍历 `request.getAttribute("studentList")` 渲染表格。
- 操作列中的"删除"按钮使用 `<button onclick="confirmDelete(...)"` 实现删除前确认。
- 顶部显示"欢迎管理员"和"退出登录"按钮。

**addStudent.jsp / editStudent.jsp**：
- 表单字段包含学号、姓名、性别（单选）、年龄（数字输入）、班级（下拉选择）。
- 使用 `request.getAttribute()` 回显错误提示和已填写数据。
- edit 页面中学号字段设置为 `readonly`，不可修改。

**error.jsp**：
- 统一错误提示页面，显示 request 域中的 `errorMsg`，提供"返回首页"和"返回登录"两个导航按钮。

### 3.2.8 CSS 样式要点

`style.css` 采用现代简洁风格：
- 全局使用 `box-sizing: border-box` 统一盒模型。
- 登录框和表单区域使用白色背景 + 圆角 + 阴影，突出视觉层次。
- 表格头部深色背景（`#34495e`），数据行 hover 效果提升交互体验。
- 按钮分类定义：主按钮（蓝色）、成功（绿色）、警告（橙色）、危险（红色）、次要（灰色）。
- 错误提示使用左侧红色边框 + 浅红背景，醒目且不突兀。

## 3.4 运行结果说明

### 3.4.1 登录页面

- 页面中央显示登录框，包含系统标题"学生信息管理系统"。
- 输入框：用户名、密码，均为必填项。
- 错误提示：登录失败时，输入框上方显示红色错误信息"账号或密码错误"。

### 3.4.2 首页（学生列表页）

- 顶部深蓝色导航栏，左侧显示"学生信息管理系统"，右侧显示"欢迎管理员"和"退出登录"按钮。
- 页面主体为白色卡片区域，标题"学生列表"右侧有绿色"添加学生"按钮。
- 数据表格列：学号、姓名、性别、年龄、班级、操作。
- 操作列每行包含"修改"（橙色）和"删除"（红色）两个按钮。
- 点击"删除"弹出 JavaScript 确认对话框。

### 3.4.3 添加/修改学生页面

- 页面布局与登录框类似，表单垂直排列。
- 学号输入框有 `maxlength="6"` 限制；年龄输入框为 `type="number"` 且 `min="1"`。
- 性别使用单选按钮（男/女）；班级使用下拉选择框（三选一）。
- 提交按钮（蓝色）和取消按钮（灰色）并排显示。
- 验证失败时，页面顶部显示错误提示，已填数据保留。

### 3.4.4 错误页面

- 居中显示错误图标和标题"操作失败"。
- 显示具体错误信息（如"未找到学号为 xxx 的学生"）。
- 提供"返回首页"和"返回登录"两个导航按钮。

## 3.5 部署说明

1. **环境准备**：安装 JDK 17+、Maven 3.8+、Tomcat 9.0+。
2. **编译打包**：在项目根目录执行 `mvn clean package`，生成 `target/SISM-1.0-SNAPSHOT.war`。
3. **部署运行**：将 WAR 文件复制到 Tomcat 的 `webapps` 目录，启动 Tomcat。
4. **访问系统**：浏览器打开 `http://localhost:8080/SISM-1.0-SNAPSHOT/login`，使用 admin / 123456 登录。

> **注意**：由于数据存储在内存中，Tomcat 重启后数据会重置为初始状态。
